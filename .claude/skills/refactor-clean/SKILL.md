# リファクタ・クリーンアップ

## ワークフロー

### Step 1: 不要コード検出

```bash
npx knip                    # 未使用ファイル、エクスポート、依存関係
npx depcheck                # 未使用 npm 依存関係
npx ts-prune                # 未使用 TypeScript エクスポート
```

ツールがなければ Grep でエクスポートのインポート数をチェック。

### Step 2: 安全度分類

| 安全度 | 例 | アクション |
|--------|---|-----------|
| **SAFE** | 未使用ユーティリティ、テストヘルパー、内部関数 | 自信を持って削除 |
| **CAUTION** | コンポーネント、API ルート、ミドルウェア | 動的インポートと外部消費者を確認 |
| **DANGER** | 設定ファイル、エントリーポイント、型定義 | 調査してから判断 |

### Step 3: 安全な削除ループ

SAFE アイテムごとに:

1. **テストスイート実行** — ベースラインを確認（全グリーン）
2. **不要コード削除** — Edit ツールで外科的に除去
3. **テスト再実行** — 何も壊れていないことを確認
4. **テスト失敗時** — `git checkout -- <file>` で即座に戻し、スキップ
5. **テスト成功時** — 次のアイテムへ

### Step 4: CAUTION アイテムの処理

削除前に:
- 動的インポートを検索: `import()`, `require()`
- 文字列参照を検索: ルート名、設定内のコンポーネント名
- 公開パッケージ API からエクスポートされていないか確認

### Step 5: 重複統合

不要コード削除後に:
- ほぼ重複する関数（80% 以上類似）→ 1 つに統合
- 冗長な型定義 → 統合
- 価値を追加しないラッパー関数 → インライン化
- 目的のない re-export → 間接参照を除去

### Step 6: レポート

```
Dead Code Cleanup
──────────────────────────────
Deleted:   12 unused functions
           3 unused files
           5 unused dependencies
Skipped:   2 items (tests failed)
Saved:     ~450 lines removed
──────────────────────────────
All tests passing ✓
```

## ルール

- テストを実行せずに削除しない
- 1 つずつ削除 — アトミックな変更でロールバックを容易に
- 不確実なら残す — デッドコードを残す方が本番を壊すよりまし
- クリーンアップ中にリファクタリングしない — 関心の分離
